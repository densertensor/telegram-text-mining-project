## 2. Три системные находки профилирования

1. **Подписные блоки и рейтинг-подборки — главный шум вне регулярных выражений.** Подписи каналов (включая канал с именем
   «РОСКОМНАДЗОР» и массовые «Подписаться на MAX») и ТОП-100-подборки дают систематические ложные совпадения,
   причём асимметрично по периодам (подписи MAX взлетели после отсечки) — то есть искажают именно динамику.
   Решение: на этапе фильтрации вырезать подписные строки до поиска совпадений и исключать
   рейтинг-листы целиком. Паттерн MAX использовать только в контекстной форме (после чистки: ~153 «до» / ~117 «после»).
2. **Безличный пласт** (корпус C выше).
3. **Пилот атрибуции вины** (пересечение «исполнители × руководство», 207 постов, шум 8%, 60% окрашенных):
   рамку «виноваты бояре» производит сам Кремль (Песков: «блокировками занимается РКН»), а патриотические
   каналы чаще **высмеивают** эту переадресацию, чем принимают; «искренний» вариант паттерна — меньшинство;
   чаще вина кладётся на «систему в целом» или оба уровня сразу. Ограждается узко Путин; Песков/Кириенко/Шадаев
   попадают в «бояре». Следствие для кодировки: направление вины — отдельная переменная
   (Путин лично / Кремль-АП / Песков / исполнители / оба уровня / система в целом / внешний объект (Дуров, Запад) /
   мета-ирония над официальной переадресацией вины).

## 3. Аутлайн работ (по лекалам президентской части)

- **Этап 1. `filter_executors_from_sqlite.py`** (по образцу `filter_putin_posts_from_sqlite.py`):
  razdel+pymorphy3, lower+ё→е; включения и гарды из §1; общая предобработка: вырезание подписных блоков,
  исключение рейтинг-листов, страновой фильтр (CLDR/Babel, расширенный), views≥1.
  Выходы: JSONL-корпуса A, B2, C (+ опц. флаги ФАС/ГП/правительство/парламент) с меткой объекта в каждой записи,
  CSV/PNG по датам с отсечкой, отчёты об исключениях (по аналогии с country_matches).
- **Этап 2. Эмбеддинги + BERTopic + сентимент-ансамбль** — единый запуск на объединённом кейс-корпусе
  (~46.5k «до» / ~5k «после» сырых; после гардов меньше) с метками объектов; модели те же, что в президентской части
  (USER-bge-m3; cardiffnlp + seara/rubert + GeRaCl zero-shot). Параметры адаптировать к меньшему корпусу
  (min_topic_size ~50, n_neighbors ~30–50 — подобрать).
- **Этап 3. До/после-анализ по объектам:** доли тональностей и Δ по объектам (A vs B2 vs C vs президентский корпус),
  сценарии v1/v2 как в президентской части; стат-тесты (χ², бутстреп долей);
  ключевой тест гипотезы — разность изменений негатива: Δneg(исполнители) − Δneg(руководство/Путин).
- **Этап 4. Кодирование направления вины** (категории из §2.3) на двух подкорпусах: пересечение
  «исполнители × руководство» (207 постов целиком, включая 106 «до») и окрашенная часть корпуса C «после».
  Это прямой тест «лестницы Норрис», не сводящийся к сентименту.
- **Этап 5. Визуализации и сводный отчёт** по лекалам президентской части (sentiment dynamics, delta share,
  before/after CSV) + сопоставление с президентскими результатами (neg 21.4%→23.4% и т.д.).

## 4. Решения (согласованы с руководителем проекта 2026-06-10)

1. Корпус C (безличный) — **строить**, оба периода («до» — после ужесточения контекстного шаблона).
2. ФАС/Генпрокуратура, Мишустин/кабмин, Володин/Матвиенко — **опциональные уровни** (GOV/PARL/AGENCIES), не в ядре.
3. Тематическая модель — **единая** на объединённом кейс-корпусе. ВАЖНО: исследование ведётся на уровне
   **крупных агрегатов** (президент / другие фреймы «политической власти» / органы-исполнители — группами,
   не по отдельным сущностям); вклад отдельных сущностей в сентимент/топики показывается дополнительным
   разрезом — ВМЕСТЕ с агрегатным анализом, а не вместо него.
4. Этап 4 (кодирование направления вины) — **LLM на Sonnet** с выборочной ручной проверкой;
   масштаб запуска подтверждается отдельно перед стартом.
